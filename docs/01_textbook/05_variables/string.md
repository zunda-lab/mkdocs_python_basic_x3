# 文字列型

## 文字列とは

プログラミングの世界では、文のように文字が連なっているものを「文字列」と言います。
文字列の定義はプログラミング言語によって、微妙に異なるのですが、Pythonの場合は、いくつかの文字を「引用符」で囲むことで、文字列を定義します。
引用符？引用符ってなんじゃい。そうですね。日常生活ではあまり使わない単語ですよね。
次のスライドで詳しく説明します。

## 引用符とは

引用符とは、ダブルクォーテーションまたはシングルクォーテーションのこと。
スライド１行目はダブルクォーテーションで囲まれた「ダブルクォーテーション」という文字列、２行目はシングルクォーテーションで囲まれた「シングルクォーテーション」という文字列ということになります。逆に１行目をシングルクォーテーション、２行目をダブルクォーテーションで囲っても文字列として成り立ちます。
前のスライドで「文字列は引用符で囲む」と説明しました。言い換えると「文字列はダブルクォーテーションまたはシングルクォーテーションで囲む」ということになりますね。蛍光ペンで、ここからここまでってマーキングするイメージです。
ちなみに、キーボードでダブルクォーテーションは、シフトキーを押しながら数字の２のキー、シングルクォーテーションはシフトキーを押しながら数字の７のキーを押して入力します。

## 引用符のルール

次は、引用符で文字列を定義する場合のルールをいくつか説明します。
１つ目！文字列は同じ種類の引用符で囲む！つまり、ダブルクォーテーションで始めたらダブルクォーテーションで終わる、シングルクォーテーションで始めたらシングルクォーテーションで終わるってことですね。このルールを守らないと、文法エラーになっちゃって、プログラムを実行することができませーん。
２つ目！文字列中にダブルクォーテーションがある場合は、シングルクォーテーションで囲む！理由は、文字列中にダブルクォーテーションがある場合、ダブルクォーテーションで囲ってしまうと、ダブルクォーテーションが３つとになり、Pythonは文字列の終わりを判断できないからです。
３つ目！文字列中にシングルクォーテーションがある場合は、ダブルクォーテーションで囲む！理由は２つ目と同じですね。文字列を定義するたび、どっちにしようかなと迷うのは時間の無駄なので、基本をどっちにするか決めることをオススメします。どっちでもいいので、決めの問題です。ちなみに、このコースでの文字列の定義は、シングルクォーテーションでの囲みに統一することにします。

## エスケープシーケンスとは

最後にエスケープシーケンスについて、少し触れたいと思います。
ひぇー、難しい言葉出てきたー。でも「エスケープシーケンスってねー」って話せるようになるとかっこいいでので是非覚えてください。
「エスケープシーケンス」は文字列の中で「特別な文字」を表す記号のことです。
この特別な文字、目に見えないものが多いんです。代表的なのは「改行」や「タブ」。プログラミングでは「改行」もひとつの文字として扱うんです。
エスケープシーケンスで扱う「特別な文字」はいくつかありますが、本コースで説明するのはそのうち一つ「改行」だけです。
改行をエスケープシーケンスで表現すると「バックスラッシュ・エヌ」となります。Windowsではバックスラッシュが入力できないので、「円マーク・エヌ」と読み替えてください。
この例のように文字列の中に「バックスラッシュ・エヌ」を入れた文字列をprint関数で画面に表示した場合は、１行目という文字列の後、改行され、２行目という文字列が表示されるというわけです。
文字列の中に改行を含める方法は何種類かあるのですが、一番多く利用されているのが、このエスケープシーケンスを利用する方法です。
エスケープシーケンスについては、この改行だけを覚えておいてください。

## バックスラッシュや円マークの入力方法

エスケープシーケンスにはバックスラッシュや円マークを使用することを前のスライドで説明しました。
円マークは金額の入力で打ち込んだことがあると思いますが、バックスラッシュって日常のパソコン操作では滅多に入力しない記号ですよね。
じゃあ、バックスラッシュをどうやって入力するかをこのスライドで伝授したいと思います。
Macの場合だけですが、キーボードでバックスラッシュは、optionキーを押しながら円マークが書いてあるキーを押して入力します。
Windowsの場合の円マークについては、知っていると思いますが、修飾キーは何も押さずに円マークが書いてあるキーを押して入力します。
