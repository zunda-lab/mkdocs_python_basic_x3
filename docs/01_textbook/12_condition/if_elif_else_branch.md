# if文（３段以上に分岐させる）

奇数偶数の判定のレクチャーで、条件分岐の基本を説明しました。奇数か偶数かという２つの分岐でしたが、このレクチャーでは3つ以上の分岐にチャレンジしましょう。

「先生、ランクってなんですか？」そうですよね。そう思いますよね。

どうしてタイトルが「点数によってランクを決定する方法」なのかは、このレクチャー最後に紹介します。

分岐条件のバリーションで３つ目のパターンとして、ご覧の「満たさないとき次の条件判定」のフローチャートを紹介しました。

一つ目の菱形では、条件を満たすときはAという処理を実行しますが、条件を満たさないときは、右下、二つ目の菱形の分岐条件を判定します。二つ目の判定で、条件を満たすときはBという処理を実行して、条件を満たさないときはCという処理を実行するパターンです。

このパターンは「3段以上に分岐するパターン」で、二つ目の菱形を増やすことで４段以上の分岐もできるんです。川に滝があってその先にまた滝があって、その先にまた滝があって、という感じになってきましたね。

このレクチャーではこの３段以上に分岐する方法に注目してみましょう。

## if文（満たさないとき次の条件判定）

次は「満たさないとき次の条件判定」パターンのif文を説明します。

まずはフローチャートです。左の菱形とifブロック、一番右のelseブロックは前のレクチャーで説明しましたので、今回は、真ん中の菱形に注目してみます。1つ目の菱形で条件を満たさなかったとき、真ん中の菱形の条件式を判定します。条件を満たすという判定であればTrueと書かれた方向に処理が進み「elifブロック」に入ります。条件を満たさなかった場合にはFalseの方向に処理が進み「elseブロック」を処理します。

では書式を説明しましょう。

```
if 条件式:
    ifブロック
elif 条件式:
    elifブロック
else:
    elseブロック
```

ここでも真ん中の部分に注目します。フローチャートの真ん中の菱形に相当する部分をelif文で記載します。elifとは「else if」の省略形です。条件式の後にはコロンが必要です。条件式が真つまりTrueだった場合の処理をelifブロックとして書きます。elifブロックもifブロック同様、先頭にスペース4つ開けます。

```
01: point␣=␣75
02: if␣point␣>=␣80:
03: ␣␣␣␣print('優秀ランク')
04: elif␣point␣>=␣50:
05: ␣␣␣␣print('普通ランク')
06: else:
07: ␣␣␣␣print('残念ランク')
```

このスライドでは、前のスライドで説明した書式のプログラム例を紹介します。説明しやすいように行の番号を書きました。

１行目で変数pointに数値75を代入しています。７５点という意味ですね。この変数は説明を簡単にするために固定の75を代入していますが、複数いる生徒の点数のように毎回違う点数が与えられるものと考えてください。

２行目と３行目はif文と条件式、そしてifブロックです。80点以上の場合、print関数で「優秀ランク」と表示していますね。

４行目と５行目は、elif文の条件式、そしてelifブロックです。80点以上でない場合、つまり80点未満の場合、このelif文に移り、50点以上の場合、print関数で「普通ランク」と表示しています。わかりやすく言うと、50点から79点までの人が「普通ランク」ということになります。５行目のelifブロックもifブロック、elseブロックと同様４つのスペースでインデントしています。

最後、６行目と７行目は、else文、そしてelseブロックです。elifの条件式を満たさなかった場合、つまり50点未満だった場合は、print関数によって「残念ランク」と表示しています。

このスライドのプログラム例がレクチャーのタイトル「点数によってランクを決定する方法」だったんですね。
