# ブロックとインデント

前のスライドで、ifブロック、elseブロックが登場しました。

条件を満たしたときに実行されるのがifブロック、満たさないときに実行されるのがelseブロックです。

ブロックには複数の行が書けます。

条件を満たしたとき１つのことしか実行できなければ不便ですよね。あれもやってこれもやってとプログラムに指示することの方が多いと思います。

ブロックはインデントを使って表します。

インデントがないとブロックはエラーとなります。

行の先頭に一律同じ幅の空白を持たせることを「インデント」と言います。

```
if 条件式:
    xxxxxxxxxx
    xxxxxxxxxx
else:
    xxxxxxxxxx
    xxxxxxxxxx
```

if条件式コロンの次の行に、チョメチョメの行が２行、elseコロンの次の行にもチョメチョメの行が２行があります。この複数行のチョメチョメには複数の文が入ります。このスライドでは２行ですが、２行とはかぎらず、１行でも１０行でも構いません。この部分をブロックと言います。

ブロックの先頭には４つの半角スペースを書きます。全角のスペースを入れるとエラーとなります。全角スペースの混入は発見しずらいので注意してください。Pythonは行の先頭に同じ数の半角スペースを入れることで、ブロックと判断するんです。この揃った字下げをインデントと言います。

実は、インデントのための半角スペースは、１文字の半角スペースやタブでも構わないのですが、Pythonを提供している団体から「インデントは４つの半角スペースとしてほしい」とオススメされています。多くのエンジニアはこの推奨に従い、半角スペース４つを使ってインデントとしています。本コースでも一律半角スペース４つでインデントします。
