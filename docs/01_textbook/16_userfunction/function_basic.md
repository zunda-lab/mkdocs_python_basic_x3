# 関数とは

関数とは、一連の処理の集まりで、呼び出すことで処理をまとめて実行してくれるものです。

これまでのミッションで一番使ってきた関数はprint関数です。print関数は、丸カッコの中で指定した文字列や数字を、ディスプレイに表示して、改行してくれる関数ですね。指定した文字や数字の表示と改行までを、まとめて実行してくれるというわけです。

関数とは「複数の文をまとめて名前をつけたもの」ということもできます。オリジナルの関数をつくると、そのことが実感できますよ。

## 関数のイメージ

関数を実社会でイメージしてみましょう。

女性社員が男性社員に「売上報告書を作ってください」と依頼しています。

男性社員は「作る手順知っていますから大丈夫です」と引き受けます。

女性社員は男性社員に売上報告書を作る材料として「伝票」を渡します。

男性社員は伝票を受け取り、手順に則り作業して「売上報告書」を作成し、女性社員に渡します。

この例を関数に例えると、手順を知っている男性社員が「関数」ということになります。

関数には入力情報として「伝票」が渡され、結果として「売上報告書」が返却されたんですね。

男性社員が、一連の処理の集まりを実行してくれることを知っているので、女性社員はそれを依頼することができるというわけです。

次は、これまで使ってきた関数で、関数をイメージしてみましょう。

連番のリストを作るrange関数で説明します。range関数は丸カッコの中に、連番の最初の数字と、最後の数字にプラス１した数字２つを指定します。そして、range関数が生成した連番のリストをイコールで変数に代入します。この例の場合は num_list 変数に代入しています。

この関数の動作イメージを絵にしてみると、range関数が準備されてあって、そこに材料として数字の１と４を投入する、そうすると関数が頑張って処理をして、結果として連番のリストを返してくれるので、それを変数に代入するという流れになります。

後で詳しく説明しますが、入力情報を「引数」、出力情報を「戻り値」と言います。

## 関数のバリエーション

次に、関数のバリエーションを４つ紹介します。

１つ目は、「引数なし・戻り値なし」パターンです。実際このパターンの関数は少ないのですが、初心者初めての関数作成では、このパターンの関数を作ってみます。

２つ目は、「引数あり・戻り値なし」パターンです。材料だけもらって、アウトプットはなしというパターンです。何かしら動作させて終わりということですね。

３つ目は、「引数なし・戻り値あり」パターンです。例えば今日の日付を取得するtoday関数などはこのパターンに分類されます。

４つ目は、「引数あり・戻り値あり」パターンです。最も多いパターンです。材料を与え、アウトプットを期待するというものです。

少し後になりますが、この４つのバリエーションごとに、関数を定義する方法を説明します。

## ２種類の関数

Pythonの関数を分類すると、大きく２種類の関数があると言えます。

１つ目は「組み込み関数」で、元々、Pythonに用意されている関数です。これまで使ってきたprint関数やrenge関数などですね。標準ライブラリとも言われ、Pythonには膨大な量の組み込み関数が用意されています。

２つ目は「オリジナルの関数」で、自分で定義して使う関数です。毎回同じ処理を記述するかわりに、関数として定義しておくことで、関数を呼び出すだけで決まった処理を繰り返し実行することができます。ユーザ定義関数などとも言われます。
