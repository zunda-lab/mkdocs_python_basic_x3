# プログラムの流れ

コンピュータに対する命令を並べたのが「プログラム」です。コンピュータは基本的にプログラムに書いてある命令を、上から下に向かって実行しますが、それはあくまでも基本であって、そうじゃない流れもあるんです。

このスライドでは、プログラムの流れ、命令がどのような順序で実行されているかを説明しましょう。

代表的なプログラムの流れは、ご覧の「順次」「分岐」「繰り返し」の３種類。

「順次」は、命令を上から下に向かって実行します。一つ目の命令が終了したら、一つ下の命令を実行するということですね。

「分岐」は、こういう場合はこっちの命令、そうでない場合はこっちの命令というように、条件によって、違う命令を実行するという流れです。条件を判定するのも命令なんです。判断結果がAであれば左の命令、Bであれば右の命令を実行するというイメージですね。我々の世界でもありすよね。例えば、テストの点数が５０点以上だったら進級、５０点未満だったら落第とか。これが「分岐」です。

最後は「繰り返し」。図を見てください。上の命令が終了したら下の命令を実行しますが、下の命令が終了したら、もう一度同じ命令を実行するという流れです。何回か、繰り返すということですね。何回繰り返すかは、時と場合によるんですけど、永遠に繰り返すと終わらなくなってしまうので、条件を決めて、その条件を満たしたら終了するようにします。下の命令は矢印がループになって囲んでいるので「繰り返し」は「ループ」って言ったりもします。
