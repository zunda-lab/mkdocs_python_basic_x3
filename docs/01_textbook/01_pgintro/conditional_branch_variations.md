# 条件分岐のバリエーション

このスライドでは分岐のバリエーションを３つ紹介します。

一つ目は、条件を満たすときだけ処理をして、条件を満たさないときは何も実行しないパターン。

菱形は分岐条件で、赤いチェックマークの方向が条件を満たすとき、赤いバツ印の方向が条件を満たさないときの流れです。条件が満たすときはAという処理を実行するけど、条件を満たさないときは何もせず、それ以降のプログラムを実行するという感じです。

二つ目は、条件を満たすときも、満たさない場合も処理を実行するパターン。

条件を満たすときはAという処理を実行して、条件を満たさないときはBという処理を実行するパターンですね。奇数偶数の判定ではこのパターンを使います。

三つ目は、条件を満たさないとき、次の条件を判定するパターン。

一つ目の菱形では、条件を満たすときはAという処理を実行しますが、条件を満たさないときは、右下、二つ目の菱形の分岐条件を判定します。二つ目の判定で、条件を満たすときはBという処理を実行して、条件を満たさないときはCという処理を実行するパターンです。このパターンは「3段以上に分岐するパターン」で、紹介した菱形が二つ、処理が３種類というパターンだけでなく、真ん中の菱形を増やす形で何段もの条件分岐を増やすことが可能です。第７ミッション「うるう年かどうか判定せよ」では、このパターンを使います。
