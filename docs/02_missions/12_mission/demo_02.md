# デモ（2）

それでは、これから「ファイルへ書き込む方法」のデモを行います。

「ファイルへ書き込もう」のコーディング演習では、このような画面が表示されます。エディターエリアのコメント文に問題文が転記されていますので、コメント文を読みながら、操作を進めていきましょう。

途中、わからないところがあっても大丈夫、この後のレッスンでしっかり説明します。ご安心ください。

まずは２行目を確認します。リストに果物の英単語を順不同で格納し、変数 fruits_list に定義しています。この果物の英単語をファイルに書き込みます。

次に４行目を確認します。書き込むファイルのファイル名を変数 filename に格納しています。作成するファイル名は「output.txt」ですね。

５行目と６行目のコメントには「ファイル output.txt を書き込みモードで開き、戻り値のファイルオブジェクトを変数 f に格納します。」と書かれています。７行目のテンテンテンを削除して「シングルクォーテーション・w・シングルクォーテーション」と入力します。

８行目から１０行目のコメントには「リストが格納された変数 fruits_list の要素を、カンマを間にはさんで連結し、連結した文字列を変数 s に格納します。」と書かれています。１１行目のテンテンテンを削除して「ピリオド・join」と入力します。

１２行目のコメントには「変数 s に格納された連結した文字列をファイルに書き込みます。」と書かれています。１３行目のテンテンテンを削除して「ピリオド・write」と入力します。

最後に１５行目を確認します。closeメソッドでファイルを閉じています。

これで完了です。正解かどうか確認しましょう。

「回答を確認する」ボタンをクリックします。

「正解です」が表示されました。

本当に英単語の名前が書かれたファイルが作成されたのか確認しましょう。

作成されたファイルを確認するためには、第１１ミッションで習得したファイルを読み込むテクニックを使用します。

まずはファイルを読み込みモードでオープンします。

１６行目にカーソルを移動して「f・スペース・イコール・スペース・open・丸カッコ・filename・カンマ・スペース・シングルクォーテーション・r・シングルクォーテーション・丸カッコ閉じ」と入力します。

改行して「s・スペース・イコール・スペース・f・ピリオド・read・丸カッコ・丸カッコ閉じ」と入力します。

改行して「print・丸カッコ・s・丸カッコ閉じ」と入力します。

改行して「f・ピリオド・close・丸カッコ・丸カッコ閉じ」と入力します。

「回答を確認する」ボタンをクリックします。

「答えが間違っています」となりましたが、アウトプットエリアにファイルから読み込んだ５つの果物の名前がカンマ区切りで表示されました。プログラムがちゃんと動いている証拠です。

最後に追加した４行を削除します。

「回答を確認する」ボタンをクリックします。

「正解です」に戻りました。
