# コメントについて学ぼう

プログラムは一度作ったら終わりということはありません。
バグを見つけたら直す必要がありますし、ちょっとした機能追加をしたくなるかもしれません。
あとで「ここの命令はどういう意味だったかなぁ」などと考えるようなプログラムは、修正がしにくいものです。
このレクチャーでは、修正がしやすいプログラムを書くテクニック「コメント」について学びます。

## コメントとは

みなさんは「コメント」という言葉にどのような印象をお持ちですか？「事件についてコメントをお願いします」とかで使う言葉ですよね。
コメントとは、プログラムの中に書き込むことができるメモ書きです。プログラムの説明を書く場合などに使用します。もちろん、日本語での記述も可能です。
プログラムが実行されるとき、コメントは無視されるので、動作には全く影響しません。
コメントを活用すると、コードがとても読みやすくなります。「俺のプログラムはノーコメント」なんて自慢になりませんので、ご注意を。

## コメントの書き方（単一行）

コメントは１行のコメントとするか、複数行のコメントにするかで書式が異なります。
はじめにスライドで、単一行のコメントの書き方を説明します。
プログラム中に#を書くと、そこから行末までは「コメント文」になります。

```
# コメント文
```

プログラム例を見てください。１行目がコメントです。

```
# メッセージを画面に表示します。
print('はじめてのPython')
```

このプログラムを実行させると、コメントは動作に全く影響しないので、print関数だけが実行されます。

```
はじめてのPython
（改行）
```

## コメントの書き方（複数行）

次は、複数行のコメントの書き方を説明します。
複数行のコメントは、3つのダブルクォーテーションで始まり、コメント文を複数行書いて、再び3つのダブルクォーテーションで終了します。

```
"""
コメント文
：
"""
```

ダブルクォーテーションの代わりにシングルクォーテーション3つでもOKです。
プログラム例を見てください。１行目から４行目がコメントです。

```
"""
メッセージを
画面に表示します。
"""
print('はじめてのPython')
```

単一行のコメントと同じで、このプログラムを実行させると、print関数だけが実行されます。コメントは動作に全く影響しないんです。

```
はじめてのPython
（改行）
```

以上「コメントについて学ぼう」でした。
